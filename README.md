## 项目概述

程序启动器是一款基于PyQt5开发的桌面应用程序，旨在为用户提供一个高效、美观且可定制化的程序管理工具。它允许用户将常用的应用程序、脚本和文档分类整理，并通过简洁的界面快速启动。

### 适用人群

- **普通用户**：想要简化日常应用程序访问
- **IT专业人士**：需要管理大量工具和脚本
- **设计师/开发者**：需要快速访问设计工具和开发环境
- **办公人员**：需要组织各种办公软件和文档
- **游戏玩家**：想要整理游戏快捷方式
- **系统管理员**：需要管理多台计算机上的工具集

## 功能特点

### 1. 核心功能

- **分组管理**：将程序按类别组织到不同分组
- **快速启动**：一键启动应用程序、文档或脚本
- **自定义图标**：支持使用程序自带图标或自定义图标
- **搜索功能**：快速查找分组或程序
- **收藏功能**：将常用项目和分组置顶

### 2. 高级功能

- **启动参数设置**：为程序配置启动参数
- **工作目录设置**：指定程序运行时的工作目录
- **管理员权限运行**：需要提升权限的程序
- **批量操作**：同时对多个程序进行移动或删除
- **自动备份**：每小时自动备份数据库
- **剪贴板集成**：从剪贴板快速添加程序

### 3. 技术特点

- **跨平台支持**：主要支持Windows，部分功能支持其他操作系统
- **轻量级**：占用系统资源少
- **可移植性**：无需安装，绿色运行
- **数据安全**：SQLite数据库存储，定期自动备份

## 安装与运行

### 系统要求

- **操作系统**：Windows 7/10/11 (推荐Windows 10+)
- **Python环境**：Python 3.7+
- **依赖库**：PyQt5, pypiwin32, pinyin, Pillow

### 安装方法

#### 方法一：直接运行可执行文件（推荐非技术用户）

1. 从发布页面下载最新版本的打包程序
2. 解压到任意文件夹
3. 双击运行`ProgramLauncher.exe`

#### 方法二：从源代码运行（适合开发者）

1. 确保已安装Python 3.7+
2. 克隆仓库或下载源代码
3. 安装依赖：
   ```
   pip install PyQt5 pypiwin32 pinyin Pillow
   ```
4. 运行主程序：
   ```
   python Program_Launcher.py
   ```

### 首次运行

首次启动时，程序会自动：
1. 创建数据库文件(`launcher.db`)
2. 创建备份目录(`backups`)
3. 创建图标目录(`icons`)
4. 添加一个"默认分组"

## 基础使用指南

### 1. 界面概览

程序主界面分为以下几个部分：
- **顶部搜索栏**：快速查找分组或按钮
- **左侧标签页**：显示所有分组
- **主内容区**：显示当前分组的程序按钮
- **底部控制按钮**：添加分组/按钮、批量操作等

### 2. 添加第一个程序

1. 点击"添加按钮"
2. 在弹出的对话框中：
   - 输入按钮名称（如"Chrome浏览器"）
   - 点击"浏览"选择程序路径（如`C:\Program Files\Google\Chrome\Application\chrome.exe`）
   - （可选）设置启动参数、工作目录等
   - 点击"保存"

### 3. 基本操作

- **启动程序**：点击对应按钮
- **编辑项目**：右键点击按钮选择"编辑"
- **删除项目**：右键点击按钮选择"删除"
- **切换分组**：点击左侧不同的标签页
- **搜索程序**：在顶部搜索栏输入关键词

## 高级功能详解

### 1. 分组管理

#### 创建分组

1. 点击"添加分组"按钮
2. 输入分组名称（如"办公软件"、"开发工具"等）
3. （可选）勾选"收藏分组"使其置顶
4. 点击"保存"

#### 分组操作

- **重命名分组**：点击分组标签旁的"编辑"按钮
- **删除分组**：点击分组标签旁的"删除"按钮（将删除组内所有程序）
- **调整顺序**：拖动标签页到新位置
- **收藏分组**：在编辑对话框中勾选"收藏分组"

### 2. 程序按钮高级设置

#### 启动参数

某些程序支持启动参数来控制其行为。例如：
- Chrome浏览器：`--incognito`（隐身模式）
- Visual Studio Code：`-n`（新窗口）
- WinRAR：`-p密码`（带密码解压）

#### 工作目录

指定程序启动时的当前工作目录。例如：
- 命令行工具：设置为脚本所在目录
- 资源管理器：设置为常用文件夹路径

#### 管理员权限

对于需要提升权限的程序（如注册表编辑器、系统配置工具等），勾选"以管理员权限运行"选项。

#### 自定义图标

程序会自动提取.exe文件的图标，也可以：
1. 点击"设置图标"按钮选择自定义图标文件(.ico/.png)
2. 或点击"清除"恢复默认图标

### 3. 批量操作模式

1. 点击"批量操作"按钮进入批量模式（按钮变为红色）
2. 点击要选择的程序按钮（选中按钮会高亮显示）
3. 右键点击任意按钮选择批量操作：
   - 批量移动：将选中按钮移动到其他分组
   - 批量删除：删除所有选中按钮
4. 再次点击"批量操作"按钮退出该模式

### 4. 剪贴板快速添加

当剪贴板中包含可执行文件路径时：
1. 按`Ctrl+V`或右键粘贴
2. 程序会自动检测并弹出添加对话框
3. 自动填充路径和名称
4. （可选）自动提取程序图标

### 5. 数据库备份与恢复

程序每小时自动备份数据库到`backups`目录，保留最近5个备份。如需手动恢复：
1. 关闭程序
2. 从`backups`目录复制所需备份文件
3. 重命名为`launcher.db`替换原文件
4. 重新启动程序

## 专业用户指南

### 1. 数据库结构

程序使用SQLite数据库存储数据，主要表结构：

#### groups表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | TEXT | 分组名称 |
| position | INTEGER | 排序位置 |
| is_favorite | INTEGER | 是否收藏 |

#### buttons表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| group_id | INTEGER | 所属分组ID |
| name | TEXT | 按钮名称 |
| path | TEXT | 程序路径 |
| arguments | TEXT | 启动参数 |
| working_dir | TEXT | 工作目录 |
| run_as_admin | INTEGER | 是否管理员权限 |
| icon_path | TEXT | 图标路径 |
| position | INTEGER | 排序位置 |
| is_favorite | INTEGER | 是否收藏 |

### 2. 高级配置

#### 修改默认设置

通过编辑源代码可以修改：
- 默认颜色主题（修改`MacaronColors`类）
- 按钮大小（在`add_group_tab`方法中修改）
- 备份频率（修改`backup_timer`间隔）

#### 扩展功能

开发者可以通过以下方式扩展功能：
1. 添加新的颜色主题
2. 实现插件系统
3. 增加网络同步功能
4. 添加多语言支持

### 3. 命令行参数

程序支持以下命令行参数：
- `--reset`：重置所有设置（开发中）
- `--debug`：启用调试模式（开发中）

## 应用场景举例

### 1. 办公自动化

**场景**：财务人员每天需要使用多个财务软件和Excel表格

**解决方案**：
1. 创建"财务工作"分组
2. 添加：
   - 财务软件快捷方式
   - 常用Excel模板文件
   - 日报生成脚本
3. 设置工作目录到财务数据文件夹
4. 将分组收藏置顶

### 2. 开发环境

**场景**：开发者需要快速访问各种开发工具和项目

**解决方案**：
1. 创建分组："IDE"、"版本控制"、"数据库"等
2. 添加：
   - Visual Studio Code（带项目路径参数）
   - Git Bash（设置工作目录到代码仓库）
   - 数据库管理工具
   - 常用文档
3. 使用批量操作整理工具分类

### 3. 游戏娱乐

**场景**：游戏玩家想要整理多个平台的游戏

**解决方案**：
1. 按平台创建分组："Steam"、"Epic"、"本地游戏"
2. 为每个游戏：
   - 添加快捷方式
   - 设置自定义图标（游戏logo）
   - 添加启动参数（如跳过启动动画）
3. 使用搜索功能快速找到游戏

### 4. 系统管理

**场景**：IT管理员需要管理多台服务器

**解决方案**：
1. 创建分组："远程管理"、"监控工具"、"脚本"
2. 添加：
   - 远程桌面连接（带服务器IP参数）
   - 管理控制台（设置管理员权限）
   - 维护脚本
3. 使用收藏功能标记常用工具

## 常见问题解答

### Q1: 程序无法启动某些应用程序

**可能原因**：
- 路径中包含空格但未正确引用
- 缺少必要的运行时环境
- 权限不足

**解决方案**：
1. 确保路径正确并用引号括起来
2. 检查目标程序是否依赖特定环境（如.NET Framework）
3. 对于系统工具，勾选"以管理员权限运行"

### Q2: 图标显示不正确

**可能原因**：
- 图标文件被移动或删除
- 不支持的图标格式

**解决方案**：
1. 重新设置图标
2. 使用.ico或.png格式的图标
3. 确保图标文件与程序在同一驱动器上（Windows限制）

### Q3: 数据库损坏怎么办

**解决方案**：
1. 关闭程序
2. 从`backups`目录恢复最近的备份
3. 或删除`launcher.db`让程序重新创建（会丢失数据）

### Q4: 如何迁移到新电脑

**步骤**：
1. 复制`launcher.db`和`icons`文件夹到新电脑
2. 确保程序路径在新电脑上有效
3. 或导出/导入功能（开发中）

### Q5: 能否添加网站快捷方式

**解决方案**：
1. 创建浏览器快捷方式（如chrome.exe）
2. 设置URL为启动参数（如`https://example.com`）
3. 设置自定义图标为网站favicon
